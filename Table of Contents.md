## 1. 🏗️ Фундамент (The Base)

### 1.1 Лицензирование и Свобода (LFS)
*   **GNU GPL (General Public License)** — «Вирусная» свобода, обязывающая производные работы оставаться открытыми. [инфо]()
*   **MIT License** — Максимально пермиссивная лицензия, разрешающая любое использование кода. [инфо]()
*   **Apache License 2.0** — Лицензия с явным предоставлением патентных прав пользователям. [инфо]()
*   **BSD Licenses** — Семейство пермиссивных лицензий, дающих максимум свободы. [инфо]()
*   **Copyleft vs Copyright** — Концепция «авторского права наоборот» в противовес традиционному праву. [инфо]()
*   **Creative Commons (CC)** — Стандарт лицензирования документации и медиа-контента. [инфо]()

### 1.2 Операционные системы (OS)
**Универсальные и серверные дистрибутивы / General Purpose & Server OS**
*   **Debian** — Индустриальный стандарт стабильности и предсказуемости. [инфо]()
*   **Ubuntu Server** — Самый популярный дистрибутив с широчайшей поддержкой сообщества. [инфо]()
*   **Red Hat Enterprise Linux (RHEL)** — Эталон корпоративной ОС с жестким циклом поддержки. [инфо]()
*   **Rocky Linux** — Свободный бинарно-совместимый форк RHEL (замена CentOS). [инфо]()
*   **AlmaLinux** — Корпоративная альтернатива RHEL с открытым исходным кодом. [инфо]()
**Специализированные и легкие системы / Specialized & Lightweight**
*   **Arch Linux** — Система для глубокого контроля и понимания внутренних механизмов ОС. [инфо]()
*   **Alpine Linux** — Сверхлегкая ОС (5МБ) на базе musl и BusyBox для контейнеров. [инфо]()
*   **Fedora Server** — Инновационный полигон для обкатки технологий RHEL-стека. [инфо]()
**Защищенные бастионы и Unix-наследие / Security & Unix Heritage**
*   **FreeBSD** — Высокопроизводительная ОС с нативными подсистемами ZFS и Jails. [инфо]()
*   **OpenBSD** — Бастион безопасности с бескомпромиссным подходом к аудиту кода. [инфо]()
*   **illumos** — Прямой наследник OpenSolaris (ZFS, DTrace, Zones). [инфо]()
*   **GNU/Hurd** — Микроядерная архитектура, представляющая академический интерес. [инфо]()

### 1.3 Сетевая связность и анализ (NAC)
**Анализ трафика и диагностика / Traffic Analysis & Diagnostics**
*   **Wireshark** — Графический анализатор протоколов для детального разбора трафика. [инфо]()
*   **Tcpdump** — Консольная утилита для захвата и анализа пакетов на удаленных серверах. [инфо]()
*   **MTR (My Traceroute)** — Диагностика сети, объединяющая функционал ping и traceroute. [инфо]()
**Аудит безопасности и сканирование / Security Audit & Scanning**
*   **Nmap** — Мощный сканер сети для обнаружения хостов и открытых портов. [инфо]()
*   **Netcat (nc)** — Универсальная утилита («швейцарский нож») для работы с TCP/UDP. [инфо]()
**Связанность и туннелирование / Connectivity & Tunneling**
*   **WireGuard** — Современный высокоскоростной протокол VPN с криптографией. [инфо]()
*   **OpenVPN** — Классическое решение для создания зашифрованных каналов связи. [инфо]()
*   **Tailscale** — Решение для создания Mesh-сетей (SDN) на базе WireGuard. [инфо]()
*   **ZeroTier** — Платформа для виртуализации глобальных сетей (P2P SDN). [инфо]()
**Балансировка и проксирование / Balancing & Proxying**
*   **HAProxy** — Высокопроизводительный балансировщик нагрузки TCP/HTTP. [инфо]()
*   **Nginx** — Эффективный Reverse Proxy для терминации SSL и защиты бэкендов. [инфо]()

## 9. Децентрализованные и суверенные коммуникации

### 9.1. Технологический стек передачи медиа-данных (Audio/Video/VoIP)
*Набор протоколов, обеспечивающих установку соединения, шифрование и передачу потоков в реальном времени.*

**Протоколы (The Core Stack):**
* **WebRTC** — Стандарт для P2P-передачи видео/аудио в браузере и мобильных приложениях. Включает механизмы обхода NAT (ICE, STUN, TURN).
* **Jingle (XEP-0166)** — Расширение протокола XMPP для управления мультимедийными сессиями (звонками).
* **SRTP / ZRTP** — Протоколы безопасной передачи трафика. ZRTP позволяет сверять ключи голосом (SAS), исключая атаку MiTM.
* **SIP (Session Initiation Protocol)** — Классический протокол телефонии, часто используемый в связке с TLS для защищенной связи.

**Реализации и клиенты на базе этого стека:**

1. **На базе WebRTC (Централизованные/Self-hosted):**
   * **Jitsi Meet** (Сервер: Jitsi Videobridge; Клиенты: Веб-браузеры, мобильное приложение Jitsi).
   * **Nextcloud Talk** (Сервер: Nextcloud + HPB Signaling; Клиенты: Браузер, Talk Mobile App).
   * **Galene** — Легковесный видеосервер (на языке Go), использующий WebRTC для лекций и конференций.

2. **На базе Jingle/SRTP (Экосистема XMPP):**
   * **Conversations (Android)** — Эталонная реализация видеозвонков через XMPP.
   * **Dino (Linux)** — Нативный клиент, поддерживающий звонки через Jingle.
   * **Gajim** — Поддержка аудио/видео через плагины интеграции WebRTC/Jingle.

3. **На базе SIP/ZRTP (Защищенная телефония):**
   * **Linphone** — Кроссплатформенный клиент с открытым исходным кодом и поддержкой высшего уровня шифрования ZRTP.
   * **Baresip** — Минималистичный консольный (CLI) стек для работы с аудио и видео.


### 9.2. Федеративные протоколы (XMPP & Matrix)
*Инструменты построения распределенных сетей связи с поддержкой мультимедийных сессий.*

#### А. Экосистема XMPP (Jabber)
*Протокол с акцентом на расширяемость (XEP-0166: Jingle для медиа).*

**Клиенты с поддержкой аудио и видео (Full Media):**
* **Conversations (Android)** — Эталонный мобильный клиент. Безупречная поддержка аудио/видео звонков через Jingle и обход NAT.
* **Dino (GNU/Linux)** — Современный нативный клиент (Vala/GTK). Имеет встроенную поддержку видеозвонков и конференций (через библиотеки WebRTC).
* **Gajim (Desktop)** — Поддерживает аудио и видео звонки (требует наличия установленных GStreamer-плагинов).
* **Moxxy (Android)** — Современный клиент на Flutter с активной поддержкой мультимедиа.

**Клиенты с упором на текст (Text-Focused / CLI):**
* **Profanity (CLI)** — Консольный клиент. Аудио/видео не поддерживает (согласно философии терминала), идеален для текстовой скрытности.
* **Siskin IM (iOS)** — Поддерживает аудио/видео в рамках экосистемы Apple.

#### Б. Экосистема Matrix
*Протокол, где поддержка WebRTC является частью базового стандарта (спецификация VoIP).*

**Клиенты с поддержкой аудио и видео (Full Media):**
* **Element (Web/Desktop/Mobile)** — Флагманский клиент. Полная поддержка 1-на-1 звонков и групповых конференций через **Element Call**.
* **SchildiChat (Android/Desktop)** — Форк Element с улучшенным интерфейсом, сохраняющий полную поддержку медиа-стека Matrix.
* **FluffyChat** — Легковесный клиент (Flutter), поддерживающий аудио и видео вызовы.

**Клиенты в стадии разработки медиа-функций:**
* **Fractal (GNOME/Rust)** — Высокая безопасность памяти, но поддержка видеозвонков находится в процессе реализации (на текущий момент приоритет на текст).
* **NeoChat (KDE)** — Нативный клиент для Plasma, поддержка звонков реализована частично (зависит от версии libquotient).

#### Серверные компоненты для мультимедиа (Backend):
* **ejabberd / Prosody** — Требуют настройки внешнего **STUN/TURN** сервера (например, Coturn) для прохождения голосового трафика через брандмауэры.
* **Matrix Synapse** — Требует настройки делегирования вызовов на TURN-сервер для стабильной связи между мобильными клиентами.


### 9.3. Приватность и защита метаданных (Metadata-Free)
* **SimpleX Chat** — Мессенджер без User ID (на базе протокола SMP).
* **Session** — Работа через децентрализованную сеть луковой маршрутизации Oxen.
* **Cwtch** — Многосторонние чаты через Onion-сервисы Tor v3.

### 9.4. Автономность и Mesh-сети (Off-grid)
* **Delta Chat** — Чат на базе существующих протоколов e-mail (SMTP/IMAP).
* **Meshtastic** — Связь по радиоканалу LoRa (ячеистые сети).
* **Reticulum / Sideband** — Криптографический стек для сетей без интернета.
